## Common Mistakes to Avoid

| Mistake | Problem | Fix |
|---|---|---|
| XPath everywhere | Brittle — breaks on DOM changes | Use `getByRole` / `getByLabel` |
| Hard-coded `waitForTimeout` | Slow, non-deterministic | Use `waitForResponse` or auto-wait |
| Poor test isolation | Flaky parallel runs | Fresh context + unique test data per test |
| Shared test data | Race conditions in parallel | Scope data per test with unique IDs |
| Ignoring Trace Viewer | Slow debugging | Enable `trace: 'on-first-retry'` in CI |
| No retry strategy | One-off flakes fail the build | `retries: 2` in CI |
| Weak locator design | Fragile tests | Use accessible locators, not position-based |
| Large page objects | Hard to maintain | Split by feature, max 15 methods per class |
| Environment-specific test data | Works only on one env | Use `process.env` + seed data via API |
| No CI integration | Late feedback | Add Playwright to every PR |

## Important Concepts Checklist

- [ ] **Auto Waiting** — understand what 5 conditions are checked before every action
- [ ] **Locator Strategy** — prefer `getByRole` > `getByLabel` > `getByTestId` > CSS > XPath
- [ ] **Browser Context** — know how to create isolated sessions and reuse auth state
- [ ] **Test Isolation** — each test starts clean with its own context and data
- [ ] **Fixtures** — use for setup/teardown, dependency injection, and POM integration
- [ ] **Page Object Model** — encapsulate interactions, not assertions
- [ ] **Network Mocking** — `route.fulfill`, `route.continue`, `route.abort`
- [ ] **Authentication Reuse** — `storageState` + setup project
- [ ] **Parallel Execution** — workers, projects, sharding, isolation
- [ ] **Trace Viewer** — enable in CI, know how to read timelines
- [ ] **Reporting** — HTML, Allure, blob merge for shards
- [ ] **CI/CD** — GitHub Actions workflow with artifact upload

### Layered Architecture
```
Test Layer     — specs, assertions, test scenarios
───────────────────────────────────────────────────
Page Layer     — Page Objects, components
───────────────────────────────────────────────────
Fixture Layer  — setup, teardown, DI
───────────────────────────────────────────────────
Utility Layer  — API client, data factory, logger
───────────────────────────────────────────────────
Config Layer   — environments, credentials, timeouts
```

### Configuration Management

```typescript
// config/environments.ts
const envConfig = {
  local:   { baseURL: 'http://localhost:3000', api: 'http://localhost:8080' },
```

## Real-World Scenarios & Solutions

| Scenario | Root Cause | Solution |
|---|---|---|
| Login works locally, fails in CI | Hard-coded URLs, timing | Use env vars, enable retries |
| Element visible but click fails | Covered by overlay | Wait for overlay to disappear |
| Pass alone, fail in parallel | Shared test data | Unique IDs per test, no shared state |
| Random timeout in checkout | Race condition | `waitForResponse` after submit |
| Flaky API failures | External dependency | Mock the API with `page.route` |
| File download in Docker | No real filesystem path | `download.saveAs(path)`, use `/tmp` |
| SSO authentication | Third-party redirect | Use `storageState` from headful run |
| Visual differences across browsers | Font rendering | Per-browser baselines, allow pixel ratio |
