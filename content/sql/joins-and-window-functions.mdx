## Join semantics in one line each

- **INNER** — rows matching in both tables.
- **LEFT** — all left rows; nulls where no right match (great for "find records with no related row").
- **FULL OUTER** — everything from both, nulls where unmatched.

Classic question — *users who never placed an order*:

```sql
SELECT u.id, u.email
FROM users u
LEFT JOIN orders o ON o.user_id = u.id
WHERE o.id IS NULL;
```

## Window functions don't collapse rows

Unlike `GROUP BY`, a window keeps every row and adds a computed column over a "window" of rows.

```sql
SELECT
  name,
  department,
  salary,
  RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dept_rank
FROM employees;
```

`ROW_NUMBER` (unique), `RANK` (gaps on ties), `DENSE_RANK` (no gaps) — knowing the tie behavior difference is a frequent gotcha.

## Running totals & deduplication

```sql
-- Running total per user, ordered by date
SUM(amount) OVER (PARTITION BY user_id ORDER BY created_at)

-- Keep only the latest row per key
SELECT * FROM (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY email ORDER BY updated_at DESC) AS rn
  FROM contacts
) t WHERE rn = 1;
```
